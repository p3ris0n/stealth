# Team Digest Generator Specs

## Purpose

Generate a daily team digest that summarizes email activity, pending items, completed work, and team member contributions — all within an isolated workspace, with no dependency on the main Stealth Mail application.

## Release Scope

- Release tier: V2 later-release tool
- Audience: team
- Folder ownership: `tools/v2/team/team-digest-generator/`
- Integration status: isolated mini-product workspace

## In-Scope Behavior

- Represent individual team email activity as digest items.
- Group digest items into logical sections: new messages, pending items, completed items, team summaries.
- Preserve each item's source email metadata for reviewer traceability.
- Capture priority and attention flags for each digest item.
- Provide per-team-member contribution summaries within the digest.
- Define fixtures that cover each digest section type.
- Provide a folder-local validation path for OSS reviewers.

## Out-of-Scope Behavior

- Main app routing or dashboard registration
- Inbox ingestion or mail rendering engine changes
- Authentication, wallet, Stellar, or database changes
- Shared design system changes
- Notification delivery or real-time collaboration side effects
- Natural language generation — digest text is defined via fixture data, not generated by an LLM
- Live network calls, secrets, or production data

## Digest Item Contract

Each digest item should include:

- `id`: stable fixture-local digest item identifier
- `type`: one of `new_message`, `pending_item`, `completed_item`, `team_summary`
- `title`: short action label
- `sourceEmailId`: source email identifier or `null` for rollup items
- `teamMember`: assigned team member name or `null`
- `priority`: one of `low`, `medium`, `high`
- `timestamp`: ISO date string or `null`
- `requiresAttention`: true when the item needs human review

## Digest Contract

The top-level digest should include:

- `date`: ISO date string for the digest date
- `generatedAt`: ISO datetime when the digest was generated
- `team`: team name
- `items`: array of digest items conforming to the digest item contract
- `summary`: object with:
  - `totalItems`: total number of digest items
  - `requiresAttention`: count of items needing attention
  - `teamMembers`: array of unique team member names
